Return how the organization's recommendations have been distributed
across the last 30 days of completed scans. Useful for review-queue
sizing and reporting.

```http
GET /api/v2/org/usage/priority
```

## Response

<ResponseField name="counts" type="object" required>
  Object keyed by recommendation value. Always includes every
  recommendation even when zero, plus an `unknown` bucket for
  completed scans missing a recommendation.
</ResponseField>

<ResponseField name="total" type="integer" required>
  Sum of all bucket counts.
</ResponseField>

<ResponseField name="window_days" type="integer" required>
  Always `30`.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 UTC timestamp of when the data was generated.
</ResponseField>

<ResponseField name="source" type="string" required>
  `cache` if served from the warmer cache, `live` if computed on the
  fly.
</ResponseField>

### `counts` keys

- `no_flags`
- `review_low`
- `review_medium`
- `review_high`
- `unknown`

## Example

```bash
curl https://api.tumban.com/api/v2/org/usage/priority \
  -H "Authorization: Bearer sk_xxx"
```

```json
{
  "counts": {
    "no_flags": 812,
    "review_low": 167,
    "review_medium": 92,
    "review_high": 41,
    "unknown": 0
  },
  "total": 1112,
  "window_days": 30,
  "updated_at": "2026-04-29T12:00:00.123456+00:00",
  "source": "cache"
}
```
